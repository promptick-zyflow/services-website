# Services Website Project — Epic Requirement

> **PROPOSED — confirm or correct.** Authored by Atlas on 2026-06-12 from epic #1's body + the 5 sub-issue bodies (#2–#6).

## Problem

The promptick-zyflow services website (zyflow-landing repo) is the public face of the org — but it doesn't yet exist as a designed set of high-converting landing pages for the org's core AI services. Without it, the company has no public site to show its services, capture leads, or anchor partner/marketing conversations.

## Target user

- **Primary:** prospective customers and partners visiting the public website.
- **Secondary:** the internal team maintaining the brand's external presentation (Sahil — Discord `1508708497636724818`, the assigned lead).

## Goal

Build a complete, high-converting public services website for promptick-zyflow that showcases the org's AI solutions, personal agents, and hyper-customized deep agents, with each major service line having its own landing page.

## Scope (MVP — the 5 deliverables)

The 5 sub-issues define the MVP scope for the website. Each is a single landing page (or in #2's case, a project-transfer kickoff):

1. **#2 KT/Handover from Ankit to Sahil** — knowledge transfer, repo transfer, credentials, project context.
2. **#3 Deep Agent Landing Page for Loan Brokers/Lenders** — automation, lead qualification, document processing, compliance, deal management.
3. **#4 Project Management Deep Agent Landing Page for Businesses** — sprint planning, task tracking, resource allocation, reporting, cross-team coordination.
4. **#5 Product Manager Agent Landing Page** — requirements documentation, user story generation, roadmap planning, stakeholder management, PRD creation.
5. **#6 Content Director/Creator Deep Agent Landing Page** — content strategy, multi-channel creation, SEO, brand voice, performance analytics.

Each landing page shares a common structure: hero → features/use cases → technical/integration capabilities → pricing/demo/CTA.

## Non-goals

- This epic is the *services website* — not the "Enhance website" refresh (see #14, separate ticket).
- Backend systems beyond the static/marketing site.
- E-commerce / paid checkout flow.

## Acceptance criteria

- [ ] The site has a consistent, polished, mobile-friendly layout across all pages.
- [ ] Each of the 5 service-line landing pages is live and navigable.
- [ ] Each landing page communicates the agent's value proposition, capabilities, and a clear CTA.
- [ ] Site loads correctly on desktop and mobile browsers.
- [ ] Handover from Ankit to Sahil is complete (repo, credentials, walkthrough).

## Success metric

A polished, externally-presentable services website that lists the org's main AI services with working landing pages and clear CTAs, and that Sahil can independently maintain post-handover.

## Dependencies

- The "Enhance website" ticket (#14) — sibling epic for the visual refresh; runs in parallel or after.
- Ankit's availability for the KT session (#2).
- A consistent site-wide design system / shared layout — implied by "high-converting landing pages"; if not already present, may need a small follow-up.
